Overview

Software APIs evolve over time and customers should upgrade to the new features to use the benefits. However, compatibility with older versions should be maintained. To sustain compatibility, obsolete API elements must contain clear messages about how clients perform the updates. The used database contains 661 Java systems hosted on GitHub. For this study, we identified 12 characteristics that influence the substitution frequency of messages in systems. The characteristics are: popularity, size, community, activity and maturity. The systems were grouped according to their goal: Applications, Software Tools, Libraries and Web Frameworks, Libraries and Web Frameworks that are not Web, and Basic software (system software).

1. Popularity: Number of stars in GitHub, number of forks and number of watchers

2. Size: Number of files and number of API elements

3. Community: Number of contributors, average of files per contributor and Average of API elements per contributor

4. Activity: Number of commits, number of releases and average launch time between releases in days

5. Maturity: System age in days.
